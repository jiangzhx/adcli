# 升级版营销产品与目标设置说明

2024年11月，巨量营销升级版针对「推广产品与目标」模块进行升级，本文档将结合本次更新说明API相关改动

# 二、 API创编主要变化及影响

每个推广目的下涉及的接口传参改动，可详见此文档：「[【终版】推广产品与目标升级-Marketing API改动](https://bytedance.larkoffice.com/docx/UkVHd5Ne4onfGsxHsF4cO4xcnis)」

| 巨量营销升级版PC端改动内容 | 对API使用的主要影响 | 是否兼容？ |
| --- | --- | --- |
| 「关联产品」、「投放内容与目标」融合为「营销产品与目标」 | 获取项目列表接口新增获取“营销产品”、“投放载体”字段，便于您了解API参数设置与PC端设置的对应关系 创建项目接口改动较小，可详见本文档第三部分 | 不涉及兼容 |
| 「应用调起-直达链接」填写位置变更(仅应用营销目的) | Before：直达链接&备用直达链接在创建项目时填写 After：直达链接&备用直达链接只能在创建营销时填写 | ✅ 预计24年12月10日起该场景下，禁止在项目层级传入直达链接 |
| 「内容营销-投放产品=抖音号」抖音号填写位置变更（仅内容营销营销目的） | Before：如投放内容为抖音主页或直播场景，需在单元层级填写抖音号信息 After：满足以下条件时，可在项目层级设置抖音号ID 内容营销-通投-不投商品，可投放抖音号（非必须） 内容营销-搜索，必须在项目层级设置抖音号 | ✅ 预计24年12月10日起，左侧在项目层级必填抖音号ID场景生效 |
| 服务全面资产化,「微信小程序&微信小游戏投放」在项目中明确投放具体资产ID | Before：投放微信小程序/小游戏时，无需设置小程序资产id（micro_app_instance_id） After：投放微信小程序/小游戏时，必须设置小程序资产id（micro_app_instance_id） | ✅ 预计24年12月10日起该场景下，微信小程序/小游戏资产id必填 |
| 「字节小程序」投放，不再支持设置备用事件资产（不限营销目的） | Before：投放字节小程序时，创建项目时事件资产id（optimize_goal.asset_ids）可设置2个，第2个代表备用事件资产 After：投放字节小程序时，创建项目时事件资产id（optimize_goal.asset_ids）只能设置1个，不再支持备用事件资产 注意：该逻辑不区分营销目的，目前小程序、电商、销售线索营销目的下均可投放字节小程序，这些场景下都将不再支持设置备用事件资产 | ✅ 预计24年12月10日起该场景下，optimize_goal.asset_ids传入2个id会报错 |

# 三、 Marketing API新建项目如何表达营销产品/投放载体

| 营销目的 | 营销产品（创建项目API设置项，变动用红字标出) | 投放载体（创建项目API设置项，变动用红字标出) | 上线后针对营销产品/投放载体的表达，API创建项目接口逻辑变化 |
| --- | --- | --- | --- |
| 应用 | 应用：选择app_promotion_type （子目标：如应用调起/预约/下载）且项目层级不传related_product结构体 商品：项目层级传入related_product结构体设置项目绑定哪些商品，则表示投放产品=商品 | 仅当营销产品=商品时，AD PC端会出现载体选项且只有应用，此场景传参要求同左侧 | 不变 |
| 小程序 | 字节小程序：micro_promotion_type = BYTE_APP 且需要传入micro_app_instance_id（不传会报错） 微信小程序：micro_promotion_type = WECHAT_APP且需要传入micro_app_instance_id（预计12月10日后不传入会报错） 字节小游戏：micro_promotion_type = BYTE_GAME且需要传入micro_app_instance_id（不传会报错） 微信小游戏：micro_promotion_type = WECHAT_GAME需要传入micro_app_instance_id（预计12月10日后不传入会报错） 抖音号：micro_promotion_type = AWEME 商品：项目层级传入related_product结构体设置项目绑定哪些商品，则表示营销产品=商品 | 仅当营销产品=商品时，AD PC端会出现载体选项： 字节小程序：此场景传参要求同左侧 微信小程序：此场景传参要求同左侧 抖音号：此场景传参要求同左侧 | 营销产品/投放载体 为字节小程序/小游戏时，24年 11月12日起必须传入资产id，否则会报错 先前未绑定字节小程序资产的项目下，营销支持编辑，先前项目下不建议新建营销 投放产品/载体 为微信小程序/小游戏时，24年 12月10日后API创建项目未绑定资产会报错 其余逻辑不变 |
| 快应用 | 快应用：设置quick_app_id快应用资产id且项目层级不传related_product结构体 商品：项目层级传入related_product结构体设置项目绑定哪些商品，则表示营销产品=商品 | 仅当营销产品=商品时，AD PC端会出现载体选项且只有快应用，此场景传参要求同左侧 | 不变 |
| 线索 | 商品：项目层级可以传入related_product结构体设置项目绑定哪些商品，则表示营销产品=商品，也可不传表示不启用 无产品：项目层级不添加商品，不传related_product结构体 | 不管绑不绑定商品，都需要选择投放载体 橙子落地页：asset_type =ORANGE 自研落地页：asset_type =THIRDPARTY 抖音号：asset_type =AWEME 企业号落地页：asset_type =ENTERPRISE 字节小程序：asset_type =MICRO_APP且需要传入micro_app_instance_id（不传会报错） 微信小程序：asset_type =WECHAT_APPLET且需要传入micro_app_instance_id（不传会报错） | 投放载体 为字节小程序时，24年 11月12日后必须传入资产id，否则会报错 存量未绑定字节小程序资产的项目下，营销支持编辑，项目下不建议新建营销 营销产品/投放载体 为微信小程序时，预计24年 12月10日后API创建项目未绑定资产会报错 其余逻辑不变 |
| 电商 | 商品：项目层级可以传入related_product结构体设置项目绑定哪些商品，则表示营销产品=商品，也可不传表示不启用 无产品：项目层级不添加商品，不传related_product结构体 | 不管绑不绑定商品，都需要选择投放载体 橙子落地页：asset_type =ORANGE 自研落地页：asset_type =THIRDPARTY 字节小程序：asset_type =MICRO_APP且需要传入micro_app_instance_id（不传会报错） 微信小程序：asset_type =WECHAT_APPLET且需要传入micro_app_instance_id（预计12月10日后不传入会报错） | 同上 |
| 内容营销 | 抖音号：在项目层级设置了抖音号id时（native_setting.aweme_id) 商品：仅当marketing_goal = VIDEO_AND_IMAGE（短视频/图片）时，项目层级可以传入related_product结构体设置项目绑定哪些商品，则表示营销产品=商品 无产品：项目层级不添加商品以及抖音号id | 仅当营销产品=商品/无产品时，AD PC端会出现载体选项 抖音号：promotion_type = AWEME_HOME_PAGE 橙子落地页：promotion_type = LANDING_PAGE_LINK 且必须传入asset_type = ORANGE 自研落地页：promotion_type = LANDING_PAGE_LINK 且必须传入asset_type = THIRDPARTY 注意：当营销产品=抖音号时，无需设置载体，不需要传入promotion_type | 投放载体为橙子落地页/自研落地页时，24年 11月12日后创建项目必须设置asset_type，表达是投放橙子落地页还是自研落地页 存量投放载体 = 落地页的项目支持编辑，也支持在项目下新建营销/编辑营销 其余逻辑不变 |
