# 获取卡券详情

获取卡券详情接口

### 请求地址

[https://ad.oceanengine.com/open_api/2/clue/coupon/detail/](https://ad.oceanengine.com/open_api/2/clue/coupon/detail/)[可视化调试](https://open.oceanengine.com/tools/visual_debug.html?docId=1696710638772239)[SDK下载](https://open.oceanengine.com/labels/7/docs/1773083358100557)[MCP调用](https://open.oceanengine.com/labels/7/docs/1847394521222599)

### 请求方法

GET

### Header

| 字段 | 类型 | 描述 |
| --- | --- | --- |
| Access-Token 必填 | string | 授权access_token，可以通过[【获取Access token】](https://ad.oceanengine.com/openapi/doc/index.html?id=1696710506574848)接口获取 |

### 请求参数

| 字段 | 类型 | 描述 |
| --- | --- | --- |
| advertiser_id 必填 | number | 客户ID |
| activity_id 必填 | number | 活动ID |

### 应答字段

| 字段 | 类型 | 描述 |
| --- | --- | --- |
| code | number | 返回码,详见[【附录-返回码】](https://open.oceanengine.com/labels/7/docs/1696710760866831) |
| message | string | 返回信息,详见[【附录-返回码】](https://open.oceanengine.com/labels/7/docs/1696710760866831) |
| data | object | 返回数据 |
| activity_id | number | 活动ID |
| activity_type | string | 卡券类型，可选值: DIRECT_NEED_PHONE：直接发券，收集手机号，need_phone须为true DIRECT_NOT_NEED_PHONE ：直接发券，不收集手机号，need_phone须为false 并且 BindFormId会被忽略 |
| coupon | object | 卡券信息 |
| coupon_id | number | 卡券ID |
| title | string | 活动标题，不超过15字 |
| deliver_start | string | 活动开始时间，格式：%Y-%m-%d |
| deliver_end | string | 活动结束时间，格式：%Y-%m-%d |
| need_phone | number | 是否收集手机号 1：收集 0：不收集(默认) |
| need_sms_verify | number | 是否对手机号进行验证 1：验证 0：不验证 need_phone=0时，该字段无意义 |
| create_time | string | 创建时间，格式：%Y-%m-%d %H:%M:%S |
| update_time | string | 更新时间，格式：%Y-%m-%d %H:%M:%S |
| status | string | 活动状态: UNAUDITED：新建未审核 NORMAL：正常 PAUSE：暂停 AUDIT_DOING：审核中 AUDIT_FAIL：审核失败 OFFLINE： 已下线 DELETED：已删除 |
| global_limit | object | 总库存量限制 |
| day_limit | number | 每日投放的总库存，0表示不限制 |
| total_limit | number | 总投放库存限制，0表示不限制 |
| user_limit | object | 单个用户的库存量限制 |
| day_limit | number | 用户每日可参与库存限制，0表示不限制 |
| total_limit | number | 用户可参与总库存限制，0表示不限制 |
| resource_list | object[] | 优惠券列表 |
| resource_id | number | 优惠券ID |
| title | string | 优惠券标题，不超过15字 |
| merchant_name | string | 商户名称，不超过15字 |
| valid_days | number | 有效时间段，从领取日开始计算，最大值不超过365 设置此值后，valid_start和valid_end会被忽略，0表示不设置 |
| valid_start | string | 有效期开始时间 格式：%Y-%m-%d |
| valid_end | string | 有效期结束时间 格式：%Y-%m-%d |
| code_type | string | 券码类型，可选值: API：合作方api实时传入的券码 COMMON：通用码，固定值，不支持核销 MERCHANT：商家上传券码，需调用上传券码接口上传券码文件 PLATFORM：平台产生券码 |
| code_refer_url | string | 商家上传券码类型时，有效券码文件的url |
| code | string | 通用券码类型时设置的券码，4-30位，只支持字母、数字 |
| condition | string | 使用须知，长度不超过50字 |
| notification | string | 领取须知，长度不超过500 |
| head_image_url | string | 卡券头图地址，即优惠券背景图 |
| logo_image_url | string | 商家logo地址 |
| service_num | string | 客服电话 |
| stock | number | 优惠券的库存，0表示不限制 |
| use_type | string | 核销类型 OFFLINE：用户线下核销 LINK：用户使用时直接跳转新链接 |
| link | string | use_type为LINK时必填，跳转链接 |
| industry_type | string | 行业列表，可选值: ENTERTAINMENT：生活娱乐 FINANCIAL：互联网金融 FOOD：美食 GAME：游戏 TICKET：休闲票务 OTHER：其他 |
| store_ids | number[] | 可用门店ID列表 建议不超过500 |
| min_amount | number | 最低消费金额，单位为分，如：500 表示500分，即5元 |
| resource_type | string | 优惠券类型。详见[【附录-枚举值】](https://ad.oceanengine.com/openapi/doc/index.html?id=1696710760171535)，可选值: COMMON：通用券 DISCOUNT：折扣券 FULL：满减券 GAME：游戏礼包 PHYSICAL：实物券 |
| gift_list | object[] | 游戏礼包列表 |
| count | number | 礼品数量 |
| icon_url | string | 礼品icon地址 |
| name | string | 礼品名称，不超过12字 |
| ios_link | string | ios端下载链接 |
| android_link | string | 安卓端下载链接 |
| discount | number | 折扣券的折扣，值为1到99，如：1代表0.1折，90代表9折。 |
| max_amount | number | 优惠券类型为折扣券时的最大减免金额，单位为分，如：500 表示500分，即5元 |
| relief_amount | number | 满减券的减免金额，单位为分，如：500 表示500分，即5元 |
| audit_msg | string | 审核信息，成功时通常为空，失败时描述原因 |
| form | object | 表单信息 |
| instance_id | number | 表单ID |
| name | string | 表单名称，不超过20个字符 |
| form_type | string | 表单类型，目前不支持设置，仅支持查询时返回 可选值：NORMAL_FORM（普通落地页）、ADVANCED_CREATIVE_FORM（附加创意）、NATIVE_FORM（原生表单） |
| validate_type | string | 线索验证类型，主要指C端是否进行短信验证，表单中含有电话类型时生效 可选值：CLUE_PRIORITY（线索量优先）、VALIDITY_PRIORITY（有效性优先）、NONE_VERIFICATION（全不出）、AUTO_VERIFICATION（智能验证，历史逻辑，效果等同于全不出）、ALL_VERIFICATION（全出），详情见[【附录-青鸟表单线索验证类型】](https://openapi.bytedance.net/openapi/doc/index.html?id=528) |
| submit_text | string | 提交文案说明，如“提交表单”，文案长度不超过8个字符 |
| enable_layer | number | 是否开启分层 枚举值：0，1（0表示不启用，1表示启用） |
| layer_submit_text | string | 分层提交文案说明，分层信息在Elements中，当不开启分层时，所有分层信息无效。文案长度不超过8个字符 |
| elements | json | 表单元素 |
| element_id | number | 元素ID |
| label | string | 元素标签名称，如“姓名“等，文案长度不超过8个字符 |
| element_type | string | 元素类型 可选值：NAME（姓名）、TELEPHONE（电话）、EMAIL（邮箱）、NUMBER（数值）、SEX（性别）、DATE（日期）、CITY（城市）、TEXT（文本）、TEXTAREA（文本域）、SELECT（下拉框）、RADIO（单选框）、CHECKBOX（多选框）、CALCULATOR（计算器）；暂不支持SELECT、RADIO、CHECKBOX、CALCULATOR的创建 |
| sequence | number | 元素排序，即元素流式展示分布的先后顺序，默认按list顺序 |
| default_value | number | 元素默认值，目前已开放的元素类中，目前仅支持element_type=NUMBER，即数值类型时填写，表示默认值 |
| layer | number | 是否分层元素，和enable_layer，layer_submit_text结合用 可选值：0，1（0表示不是，1表示是） |
| allow_empty | number | 是否允许为空，默认允许 可选值：0，1（0表示不允许，1表示允许） |
| is_unique | number | 同一表单是否限制不可重复提交，默认可重复 可选值：0，1（0表示可重复，1表示唯一） |
| extend_info | json | 扩展属性 |
| count_config | json | 计数配置，包括递增/递减、计数开始值、计数文案模版 |
| count_type | string | 计数方式 可选值：COUNT_TYPE_INCREMENT（递增计数）、COUNT_TYPE_DECREMENT（递减计数） |
| prefix | string | 计数文案前缀，例如：“截止目前已有”，不超过100个字符 |
| start_num | number | 起始数值 |
| suffix | string | 计数文案后缀，例如：“名用户参加”，不超过100个字符 |
| sign_up_config | json | 报名信息配置 |
| sign_type | string | 表单报名配置展示形式 SIGN_TYPE_SCROLL_WALL（滚动墙展示）、SIGN_TYPE_SCROLL_BAR（滚动条展示） |
| success_tip | string | 提交成功后的文案提示，文案长度不超过128个字符 |
| request_id | string | 请求的日志id，唯一标识一个请求 |
