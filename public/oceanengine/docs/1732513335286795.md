# 订阅服务

针对营销数据获取，本篇为大家提供了两种切实可行的数据推送方案，帮助大家高效、稳定、近实时地获取数据，更好地运用数据给营销投放提效。

## SPI

开发者需要先在开发者管理后台[创建订阅任务](https://open.oceanengine.com/developer/admin/subscription)，当订阅的内容发生变动时，平台会请求开发者预留的回调链接，将内容变动消息告知开发者，开发者基于收到的消息后再调用API接口获取数据，减少轮询次数。

## RDS

开发者需要先在开发者管理后台[创建订阅任务](https://open.oceanengine.com/developer/admin/rds)，当订阅的内容发生变动时，平台会实时推送数据至连山RDS进行存储，开发者可以在连山平台部署相关应用进行数据读取，无需再次调用API接口。

## RDS推送、SPI推送、HTTP API拉取能力对比

|  | RDS推送 | SPI推送 | API拉取 |
| --- | --- | --- | --- |
| 效率 | 高：内容直接推送到RDS数据库，比SPI推送和API拉取处理链路更短，效率会更高 | 中：内容推送到开发者预留的回调接口 | 低：通过接口一个个拉取，因为考虑到性能和平台稳定性，有频控限制，数据拉取效率受限 |
| 时效性 | 高：推送性能完全由RDS本身决定，性能对失效性的受限较小；数据通过binlog、主动通知等方式感知数据变化，比轮询方式时效性更高 | 中 - 高：如果开发者接口性能较好，时效可以跟RDS保持一致 | 低：因为有频控的限制，所以数据拉取频率受限，数据时效性较低 |
| 频控限制 | 受RDS数据库写入性能限制 | 受开发者回调接口性能限制 | 受频控等级限制 |
| 平台稳定性 | 推送速率可控，平台稳定性强 | 各开发者访问峰谷不完全可控，为保证平台稳定性，对访问频控有严格限制。但是在高峰期，接口超时较严重 |  |
