#### 接口简介

##### 作用

客户在创建计划的时候，会在计划上填写“有效触点监测链接”和“展示监测链接”。 其中，对于“有效触点监测链接”：

| 投放版位类型 | 投放端 | “有效触点”口径定义 |
| --- | --- | --- |
| 抖音类视频投放版位的oCPM营销 | 抖音、抖音火山版、头条、西瓜、番茄 | 3秒视频播放 + 点击 |
| 穿山甲视频投放版位（如：激励视频）的oCPM营销 | 穿山甲 | 10~20秒视频播放 + 点击 |
| 其他投放位置的营销 | 其他 | 仅点击 |

通过约定指定格式的字符串，用以代表将来将要替换的字段位置，这个格式我们统称为“宏”。

##### 格式

监测链接主要由"[https://XXXX.XXX.com?](https://xxxx.xxx.com/?)"+"参数"+"CALLBACK_URL或者CALLBACK_PARAM"+其他部分组成，且总大小不得超过10K。具体格式如下：

(1)监测链接的前缀格式为：

[https://XXX.XXX.com?](https://xxx.xxx.com/?)

(2 )宏“参数”的格式为“__参数__”，其中：

##### 举例

1.客户填写在计划上的监测链接如下：

```
xxxxxxxxxx
https://mybest.custom.com/click/?promotionid=__PROMOTION_ID__&callback_url=__CALLBACK_URL__
```

这里的 “PROMOTION_ID”，就是一个宏，代表将来真实的 PROMOTION_ID 字段会在这里填写。

在头条客户端中，用户真实产生了营销点击的时候，我们会把相应的字段换成真实的营销信息，发送给客户。

2.客户收到的真实请求：

```
xxxxxxxxxx
https://mybest.custom.com/click/?promotionid=7074140945750507528&callback=EJiw267wvfQCGKf2g74ZIPD89-vIATAMOAFCIjIwMTkxMTI3MTQxMTEzMDEwMDI2MDc3MjE1MTUwNTczNTBIAQ==
```

其中下划线的部分，就是我们替换后的真实营销信息（非 ASC 字符，进行了 url encode）

#### 请求结构

##### 通信协议

为了获得更高的安全性，推荐您使用HTTPS通道。

##### 请求方法

支持HTTP GET方法发送请求，这种方式下请求参数需要包含在请求的URL中。

##### 字符编码

UTF-8
