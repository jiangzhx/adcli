# SPI回调地址设置

使用SPI服务，需要提供一个回调地址，用于接收订阅推送的消息内容

当前文档描述结构仅为创建订阅时验证回调地址时，系统要求的应答结构，实际订阅服务推送回调的数据格式可参考[【订阅服务类型】](https://open.oceanengine.com/doc/index.html?key=ad&type=api&id=1722468664901647)

# 支持的通信协议

HTTP/HTTPS 协议

# 请求方式

GET/POST 回调地址需要同时支持GET和POST请求

# Challenge 验证

创建订阅任务填写回调地址时，系统会发送challenge挑战，验证URL的归属，请求参数如下

## 请求参数

| 字段 | 类型 | 描述 |
| --- | --- | --- |
| challenge | number | 随机数 |
| event | string | 事件值，verify_webhook为Challenge验证事件 |

## 应答参数

在收到Challeng请求后，需要按照下述格式返回应答，要求response 的 Content-Type 为 application/json

| 字段 | 类型 | 描述 |
| --- | --- | --- |
| BaseResp | object |  |
| StatusCode | number | 执行状态码，200表示成功，4xx 业务异常不重试，5xx 系统异常重试 |
| StatusMessage | string | 执行结果 |
| challenge | number | 回传请求中的challenge值 |

## 应答示例

```
xxxxxxxxxx
{
"BaseResp": {
"StatusCode": 200,
"StatusMessage": "ok"
},
"challenge": 1
}
```
