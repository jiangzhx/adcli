# 事件资产下创建监测链接组

# 此接口用于在事件管理资产下创建监测链接组

事件资产与监测链接的对应关系： 非应用资产只可建一个监测链接组，应用资产下可以有N个下载链接，每个下载链接可以建N个监测链接组

# 请求地址

[https://api.oceanengine.com/open_api/2/event_manager/track_url/create/](https://api.oceanengine.com/open_api/2/event_manager/track_url/create/)[可视化调试](https://open.oceanengine.com/tools/visual_debug.html?docId=1727898582816775)[SDK下载](https://open.oceanengine.com/labels/7/docs/1773083358100557)[MCP调用](https://open.oceanengine.com/labels/7/docs/1847394521222599)

# 请求方法

POST

# Header

字段

类型

描述

Access-Token

string

授权access_token，可以通过【[获取Access token](https://open.oceanengine.com/labels/7/docs/1696710505596940)】接口获取

Content-Type

string

请求消息类型 允许值: application/json

# 请求参数

字段

类型

描述

advertiser_id

number

客户ID

assets_id

number

资产ID

download_url

string

应用下载链接，当资产为应用类型时有效且必填

track_url_groups

object[]

监测链接组信息，IOS和安卓应用可绑定多组监测链接

action_track_url

string

点击（监测链接），只允许传入1个

track_url

string

展示（监测链接），只允许传入1个

video_play_track_url

string

视频播放（监测链接），只允许传入1个

video_play_done_track_url

string

视频播完（监测链接），只允许传入1个

video_play_effective_track_url

string

视频有效播放（监测链接），只允许传入1个

track_url_group_name

string

监测链接组名称，应用型资产有效且必填

active_track_url

string

激活监测
