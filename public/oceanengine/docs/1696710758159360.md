# 频控限制

本文介绍巨量引擎开放平台频控限制规则：所有 API 都施加了频控限流策略，一共包含3层

## 一、 什么是频控

单位时间（目前均为秒级，即qps）内请求接口的次数限制。目前开放平台有3种频控限制，分别为：客户频控、开发者频控、接口总频控

| 频控类型 | 介绍 | 超限限制 |
| --- | --- | --- |
| 客户频控 | 客户单位时间内请求单接口请求次数限制 | 超限报错错误码为40130 ，客户频次先于开发者频控发生，不会占用开发者频次。 |
| 开发者频控 | 开发者单位时间内请求单接口请求次数限制 | 超限报错错误码为40110 ，可查看开发者自身频控限制范围，调整请求频次在自身频控范围内 |
| 接口总频控 | 单位时间内所有开发者请求单接口次数限制 | 超限报错错误码为40100 ，系统总频控由系统动态调整，可在开发者频控范围内重试，或错峰，避免整点或半点峰期使用。 |

qps计算时间区间为当前请求至请求前一秒时间区间内的总请求次数，非自然秒

## 二、 如何查看频控

### 2.1 客户频控

| 标签类型 | 判定标准 | 更新规则 |
| --- | --- | --- |
| 不活跃账户 | 客户在90天内没有任何活跃行为，结合消耗、项目/营销创编、余额、有无开启中计划等条件综合判断 | 根据客户行为实时变动（峰期会有15-30分钟延迟） 注意：由于[【获取不活跃账户列表】](https://open.oceanengine.com/labels/7/docs/1834548530596872)接口查询结果为T+1，会存在接口查询结果未更新，但实际投放账户已激活解除限流的情况。 |

客户频控规则拦截的请求数不占用开发者维度的频控额度

即：开发者频次限制为30qps，窗口时间内发起25次请求，其中3次请求是由于客户维度频控拦截报错，那么该窗口内已使用的开发者qps为22，还可以再增加8次请求

### 2.2 开发者频控

#### 开发者频控查询方法

开发者频控可在开发者后台-账号管理-频控管理处查看（如下图：[前往查看](https://open.oceanengine.com/developer/admin/qps)），此列表为您提供以下信息说明

| 开发者频控相关信息 | 定义 | 数据时效 |
| --- | --- | --- |
| 开发者频控配额 | 您的开发者账户当前在该接口下被分配的请求QPS配额 QPS按滑动窗口计算每秒您的开发者账户在该接口下的最大请求次数(超频错误码=40110) | 实时数据，代表您当前的真实配额 |
| 近30天频控用量 | 代表您近30天在不同接口下的平均QPS用量情况 请注意，该指标是一个基于30天平滑计算的参考值，旨在反映长期趋势，不会与您感知到的实时峰值或短期波动完全吻合 建议：实际请求时请根据返回的错误码（40110）或响应头X-RateLimit-Dimension等来判断是否已触发开发者频控限流 | 该指标非实时值，每日更新一次 |
| 频控状态 | 开发者频控状态反映了您在接口下的配额提升资格，有两种情况： 🟢 支持提频 含义：您的开发者账户表现良好，已获得更高配额资格。 操作：在“操作列”点击确认，新配额才会生效，确认后通常需等待 5分钟正式生效 🔴 不支持提频 含义：目前暂未符合提频条件，通常原因如下： 配额充足：当前配额仍有富余，无需提升 已达上限：已达当前可分配的最高配额，无法再提升 未到提频时间点：请等待系统于每月 1日 或 15日 的定期评估 | 仅每月1日、15日9点后更新周期性评估结果 |

#### 如何提升开发者频控

系统周期性评估开发者近期请求情况，符合提升条件接口则会在【开发者后台】-【账号管理】-【[频控管理](https://open.oceanengine.com/developer/admin/qps)】处展示提频入口

| 评估周期 | 每月1日、15日更新评估结果，更新时间一般为9点以后 |
| --- | --- |
| 分配规则 | 为保障平台系统安全和公平性，我们无法提供具体的评估计算公式。请您理解，该评估机制由多个维度综合计算得出，旨在为所有开发者提供一个稳定、健康的开发环境。我们将尽可能确保分配公正、持续优化。 |
| 请求建议 | 设计接口请求方案时，遵循以下规则利于评估分配qps量级 减少不活跃账户量级。优先服务有消耗或有操作行为的账户，减少不活跃账户的冗余请求 保障请求合理性。例如针对实际没有操作投放的投放账户，减缓查询频次；在做增量更新时利用接口提供筛选能力，主动筛选更新内容；接入SPI消息通知能力，被动获取系统变动通知后针对性请求。 频繁请求不活跃账户将触发系统保护机制，导致系统判断不支持提频，长期将可能降低您的开发者频控配额。 |

### 2.3 接口总频控

接口总频控是由系统动态调整，非固定值。单位时间内所有开发者请求汇总的限制，并非单独针对某一个开发者请求，权重优先级高于开发者自身频控，一般峰期时间段可能会出现总频控限制报错。

#### 如何提升接口总频控

接口总频控与您的开发者账户无关，由系统动态调整。

## 三、 频控惩罚说明

当按滑动窗口计算每分钟，您的开发者账户在该接口的总请求次数需＜开发者QPS配额*100，超过则将触发持续5分钟的报错惩罚(超频错误码=40110)，届时您可根据响应头进一步调整请求策略

```
xxxxxxxxxx
X-RateLimit-Dimension：限流维度， developer_qpm
X-RateLimit-Limit ：分钟级请求次数上限 = 开发者QPS配额*100
X-RateLimit-Remaining：开发者分钟级请求剩余用量 （此时是0）
X-RateLimit-RetryIn：当请求被开发者频控限流时，需要等待的最短时间（s）
```

频控惩罚仅在请求超过QPS限制，且在一定时间内持续高频次达到分钟级别上限时触发。触发后5分钟内所有请求将受惩罚机制影响而全部报错失败。
