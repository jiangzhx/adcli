# 创建项目

创建 巨量引擎体验版 项目 客户下可创建项目数：200

# 请求地址

[https://api.oceanengine.com/open_api/v3.0/project/create/](https://api.oceanengine.com/open_api/v3.0/project/create/)[可视化调试](https://open.oceanengine.com/tools/visual_debug.html?docId=1740868093375503)[SDK下载](https://open.oceanengine.com/labels/7/docs/1773083358100557)[MCP调用](https://open.oceanengine.com/labels/7/docs/1847394521222599)

# 请求方式

POST Request-Body

# 请求Header

| 字段 | 类型 | 描述 |
| --- | --- | --- |
| Content-Type | string | 请求消息类型，允许值：application/json |
| Access-Token | string | 授权Access-Token，获取方法见接口文档【[获取Access-Token](https://open.oceanengine.com/doc/index.html?key=ad&type=api&id=1696710505596940)】 |

# 请求参数

## 基础参数

| 字段 | 类型 | 描述 |
| --- | --- | --- |
| advertiser_id 必填 | number | 投放账户id |
| operation | string | 项目状态，允许值: ENABLE开启（默认值）,DISABLE关闭 |
| delivery_mode | string | 投放模式，允许值： MANUAL手动投放(默认值）、PROCEDURAL自动投放 自动投放支持的landing_type：APP应用、MICRO_GAME小程序、LINK销售线索投放、SHOP电商、QUICK_APP快应用 搜索无需再表达delivery_mode参数，表达不生效，可通过delivery_type选择「常规投放」或「极速智投」，「极速智投」默认为自动投放模式 当marketing_goal= LIVE直播时，已支持MANUAL手动投放、PROCEDURAL自动投放 |

| app_promotion_type 条件必填 | string | 子目标，当 landing_type = APP 有效且必填 允许值：DOWNLOAD 应用下载、LAUNCH 应用调起、RESERVE 预约下载 当delivery_mode = PROCEDURAL 时仅支持DOWNLOAD应用下载； 当marketing_goal= LIVE时，仅支持DOWNLOAD应用下载、LAUNCH 应用调起 |
| marketing_goal 必填 | string | 营销场景，允许值：VIDEO_AND_IMAGE 短视频/图片，LIVE直播 直播营销场景传参要求： 仅支持已在营销平台签署直播投放协议的账户，支持的landing_type有应用/小程序/销售线索/内容营销 landing_type = LINK && delivery_mode = PROCEDURAL 自动投放时，仅白名单内账户设置为直播LIVE |
| ad_type 必填 | string | 营销类型，允许值：ALL 通投 SEARCH 搜索 当 marketing_goal=LIVE直播时，仅支持ALL通投 仅当landing_type=APP/LINK/NATIVE_ACTION/SHOP&&delivery_mode=MANUAL时支持搜索，否则报错 当landing_type=QUICK_APP快应用 / MICRO_GAME小程序 && delivery_mode = PROCEDURAL自动投放时，仅支持ALL通投 仅当ad_type=ALL 通投时，支持inventory_type=INVENTORY_HOMED_AGGREGATE住小帮 |
| delivery_type | string | 投放类型，不传默认为NORMAL常规投放，允许值： NORMAL 常规投放（默认值） DURATION 极速智投（仅支持搜索，ad_type = SEARCH） UBX_INTELLIGENT 智能托管（仅支持UBA项目） 设置搜索「极速智投」投放类型条件限制： 应用营销目的 && 直播/短视频营销场景 销售线索营销目的 && 直播/短视频营销场景 小程序营销目的 && 直播/短视频营销场景 电商营销目的 && 短视频营销场景 设置UBA链路「智能托管」投放类型条件限制： 营销目的为小程序 营销场景为短视频&图文 投放模式为自动投放 营销产品（投放载体）为字节小游戏 优化目标&深度优化方式为 变现ROI-7R && 7日ROI系数 |
| name 必填 | string | 项目名称，长度是1-50个字（两个英文字符占1个字） |
| budget_group_id | number | 预算组ID，一个项目一生只能被一个预算组关联，项目绑定预算组ID后，不允许修改。如果预算组被删除，绑定该预算组的项目也不能再绑定新的预算组。 该功能白名单开放，如需使用请联系销售 您需要首先通过「[创建预算组](https://open.oceanengine.com/labels/34/docs/1780159359081540)」/「[获取预算组列表](https://open.oceanengine.com/labels/34/docs/1780644067911818)」接口拿到此id |
| aigc_dynamic_creative_switch | string | 是否开启AIGC动态创意，系统将根据创意生态要求生成素材探索，不传表示不开启，创建后支持修改。枚举值： ON 开启 OFF 关闭 注意： 该功能仅支持行业白名单客户使用，如需使用可咨询对接销售/运营 必须关联产品投放，才可设置该开关（即必须传入related_product结构体） 如投放短剧商品，需要开启AIGC动态创意，请确保： 关联短剧商品内包含字节小程序投放链接 短剧剧目已获取原片使用权，可通过「[查询短剧商品原片授权申请状态](https://open.oceanengine.com/labels/7/docs/1805903150040128)」接口查询授权申请状态 投放星广联投项目时，不支持开启AIGC动态创意，传入ON会报错 |
| star_task_id | number | 星广联投任务ID，仅白名单客户支持设置并投放此类项目，如有诉求可联系对接销售或运营 【注意】绑定后不支持修改 游戏行业投放星广联投时，支持的landing_type = APP/MICRO_GAME 短剧行业投放星广联投时，支持的landing_type =APP/ MICRO_GAME |
| star_auto_material_addition_switch | string | 星广联投自动优选素材开关，开关开启后，允许系统在本项目已创建完成的单元下自动新增达人投稿的最新素材。枚举值： ON 开启 OFF 关闭 注意 仅当同时满足以下条件时可设置（否则传入报错） 星广联投白名单内客户，如有诉求可联系对接销售或运营 创建项目时绑定star_task_id，支持2种任务类型，注意限制要求不同 绑定的是短剧版星广联投任务（短剧行业投放星广联投时，支持的landing_type = APP/MICRO_GAME、此时星广联投任务通过“巨量引擎工作台”共享至投放账户使用） 绑定是游戏版星广联投任务时，需要在特定白名单内，如需使用请联系对接销售或运营 仅当投放模式=自动时（delivery_mode= PROCEDURAL）可开启 项目创建后，此项设置不允许编辑 |
| star_auto_delivery_switch | string | 星广联投全自动化开关，支持编辑 暂仅支持短剧版，枚举值： ON 开启 OFF 关闭 注意： 开启星广联投全自动化开关条件 账户属于白名单（Key: star_v3_auto_task，调用[白名单查询](https://open.oceanengine.com/labels/7/docs/1763407285579840)接口时version请传入OLD_VERSION），加白请联系对接销售 landing_type = MICRO_GAME 小程序 delivery_mode = PROCEDURAL 自动投放 开启全自动化支持的营销产品+投放载体包含以下2种情况 情况1：营销产品=字节小游戏/微信小游戏，即micro_promotion_type = BYTE_GAME/WECHAT_GAME 情况2：营销产品=商品、投放载体=字节小程序/微信小程序，即必须添加商品related_product、micro_promotion_type =BYTE_APP / WECHAT_APP 开启全自动化开关时，必须注意其他传参要求： 不支持指定星广任务id（即传入star_task_id会报错)。启用全自动化代表系统将为您在项目下自动关联营销产品对应的任务，无需手动选择 不支持启用自动素材开关（即传入star_auto_delivery_switch会报错）。启用全自动化已经意味着系统将自动为您在单元下添加任务下的素材，您无需再设置自动素材 |
| rta_id | number | RTA重定向（仅适用于联合出价项目） |

## 搜索快投

自动投放/小程序链路/投放范围仅穿山甲时不支持快投，快投关键词可通过搜索关键词模块设置

| 字段 | 类型 | 描述 |
| --- | --- | --- |
| search_bid_ratio条件必填 | float | 出价系数，出价系数可通过【获取快投推荐出价系数】查询，小数点后最多两位,取值范围 [1.00,2.00] 当符合以下所有条件时填写有效 bid_type !=NO_BID && pricing = PRICING_OCPM deep_bid_type = DEEP_BID_DEFAULT 无深度优化方式 /BID_PER_ACTION 每次付费/DEEP_BID_PACING 自动优化/ROI_COEFFICIENTROI系数/FORM_BID优选表单 创建搜索蓝海项目时，不支持搜索快投 micro_promotion_type = AWEME时，不支持设置搜索快投 |
| audience_extend | string | 定向拓展，允许值：ON:开启（默认值）， OFF:关闭 不传表示开启定向拓展，若需关闭，请传入该参数并设置为OFF 创建搜索蓝海项目时，不支持搜索快投 |

## 搜索关键词

可通过[【获取推荐关键词】](https://open.oceanengine.com/labels/7/docs/1755356193295372)获取系统推荐的搜索关键词

仅通投链路下使用搜索快投关键词及搜索极速智投链路下自定义关键词时支持传入使用keywords参数设置

| 字段 | 类型 | 描述 |
| --- | --- | --- |
| keywords 条件必填 | object[] | 待添加搜索关键词列表 当ad_type = ALL时，支持传入关键词为搜索快投关键词 最多可添加1000个 当ad_type = SEARCH & delivery_type = DURATION 极速智投场景下时，支持传入关键词为自定义搜索关键词 客户拥有search_merge_auto_blue白名单时支持 最多可添加3000个 |
| word 条件必填 | string | 关键词,关键词长度不超过30，一个汉字长度计为1，一个英文字符长度计为0.5，不能包含emoji |
| bid_type 条件必填 | string | 出价类型，允许值： 搜索快投场景 FEED_TO_SEARCH 搜索快投（默认值） 极速智投场景 CUSTOM 自定义出价，仅pricing出价方式设置为非O类场景时支持且默认 WITH_PROMOTION 随项目/单元出价（仅pricing出价方式设置为O类场景时支持且默认） |
| match_type 条件必填 | string | 匹配类型。 允许值： PHRASE 短语匹配 EXTENSIVE 广泛匹配 PRECISION 精准匹配 |
| bid 条件必填 | float | 出价，取值范围：0.2至999.0 当keywords.bid_type = CUSTOM 时有效且必填 |
| auto_extend_traffic | string | 智能拓流 ，允许值： 当ad_type = SEARCH 且 delivery_type = DURATION 时有效且必填 ON 开启 OFF 关闭 当仅使用自定义关键词（未设置蓝海关键词），且关键词不为空时，支持设置为OFF 当external_action=AD_CONVERT_TYPE_CLICK_NUM 点击量时，仅支持设置为OFF |

## 商品

传入related_product.product_setting= SINGLE 启用时，表示营销产品=商品；不传related_product或当不启用商品库时，营销产品≠商品

| 字段 | 类型 | 描述 |
| --- | --- | --- |
| related_product | object | 关联产品投放相关参数 |
| product_setting 条件必填 | string | 商品库设置 允许值：SINGLE 启用sDPA、NO_MAP不启用（默认值）、MULTI_PRODUCTS多品投放new 如果不需要关联产品投放，无需传入此参数 如果您要创建landing_type = DPA动态商品投放，需要关联商品库，无需传入此参数，关联产品投放只需要传入「product_platform_id」商品库ID，其他参数无需关注 如果您需要进行单品投放（sDPA），请务必传入product_setting= SINGLE 自动投放模式下即delivery_mode =PROCEDURAL 且 营销目的landing_type =销售线索 LINK，商品信息只需要传入下方的products结构体，您可以根据需要选择传入通用版商品或升级版商品库 参数products结构体下选择一种传入： 通用版商品：传入product_platform_id +product_id 升级版商品库：传入unique_product_id （白名单能力） 其他情况下，可根据投放的商品版本（通用版/升级版）选择传入的参数，您无需关注下方的products结构体，直接选择传入通用版商品或升级版商品库参数 通用版商品：传入product_platform_id +product_id 升级版商品库：传入unique_product_id （白名单能力） 如果进行UBP多品投放则传入MULTI_PRODUCTS，单账户下仅支持10个UBP多品项目 UBP场景下关联产品通过products结构体传入，且仅支持升级版商品库 白名单说明 【注意】product_setting = MULTI_PRODUCTSUBP多品投放仅支持在白UBP多品白名单账户使用，您可通过「[查询白名单能力](https://open.oceanengine.com/docs/1763407285579840)」接口查询客户是否命中白名单 【注意】delivery_mode = PROCEDURAL自动投放 且 landing_type= QUICK_APP快应用或MICRO_GAME小程序时，仅支持网服行业投放账户传入关联产品，您可通过「[查询白名单能力](https://open.oceanengine.com/docs/1763407285579840)」接口查询客户是否命中白名单 【注意】当前升级版商品库为白名单功能，如当前客户账号没有权限，传参会报错 |
| product_platform_id 条件必填 | number | 商品库ID，当启用商品库时必填，可通过[【商品投放-查询商品库】](https://open.oceanengine.com/labels/7/docs/1696710577036288)查询，创建后不可修改 当delivery_mode选择PROCEDURAL且landing_type选择LINK时，传入报错 【商品的传参逻辑说明】 通用版商品：传入product_platform_id +product_id 如果您要创建landing_type =DPA动态商品投放，需要关联商品库，关联产品投放只需要传入「product_platform_id」商品库ID，其他参数无需关注 |
| product_id 条件必填 | string | 产品ID，当启用商品库时必填，可通过[【商品投放-获取商品列表】](https://open.oceanengine.com/labels/7/docs/1696710578078732) 查询，创建后不可修改 当delivery_mode选择PROCEDURAL且landing_type选择LINK时，传入报错 【商品的传参逻辑说明】 通用版商品：传入product_platform_id +product_id 升级版商品库：传入unique_product_id （白名单能力，如需使用请联系运营） |
| unique_product_id 条件必填 | number | 升级版商品库产品ID，可通过【[商品投放-获取线索商品列表](https://open.oceanengine.com/labels/34/docs/1779430442685440)】查询获取id（该接口下的product_id就是unique_product_id），创建后不可修改 如果投放升级版商品库，只需要传入unique_product_id |
| products 条件必填 | object[] | 产品ID列表，上限为10 特殊： UBP多品项目传入个数限制为[2,30]，且每个产品仅可支持绑定1个UBP多品项目 满足以下任一条件，则products必传： delivery_mode =PROCEDURAL 自动投放模式 且 营销目的landing_type = 销售线索 LINK时，products结构体必传 支持UBP多品项目设置products，当related_product.product_setting = MULTI_PRODUCTS多品投放时，products结构体必传 传入products结构体，根据需要选择传入通用版商品或升级版商品库参数 通用版商品：传入product_platform_id +product_id 升级版商品库：传入unique_product_id （白名单能力） |
| product_id 条件必填 | string | 产品ID 当delivery_mode选择PROCEDURAL且landing_type选择LINK时有效且必填 【商品的传参逻辑说明】 通用版商品：传入product_platform_id +product_id |
| product_platform_id 条件必填 | number | 产品库ID 允许传入同类产品库，传入不同类产品库报错 当delivery_mode选择PROCEDURAL且landing_type选择LINK时有效且必填 【商品的传参逻辑说明】 通用版商品：传入product_platform_id +product_id 如果您要创建landing_type = DPA动态商品投放，需要关联商品库，关联产品投放只需要传入「product_platform_id」商品库ID，其他参数无需关注 |
| unique_product_id 条件必填 | number | 升级版商品库产品ID，可通过【[商品投放-获取线索商品列表](https://open.oceanengine.com/labels/34/docs/1779430442685440)】查询获取id（该接口下的product_id就是unique_product_id） 如果投放升级版商品库商品，只需要传入unique_product_id UBP多品项目只支持传升级版商品库商品，因此当满足上述条件时，unique_product_id必传 |
| asset_id | number | 投放条件ID，可通过[【商品投放-获取投放条件列表】](https://open.oceanengine.com/doc/index.html?key=ad&type=api&id=1696710579122176)获取，创建后不可修改。 |

## 营销产品与投放载体
