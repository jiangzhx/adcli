# 移动应用API上报数据

通过移动应用API上报数据，可以实现头条端和客户端的信息互通，从而提高营销的优化效果，提高营销投放效率。

【跟踪应用下载】

1. 为什么要做转化跟踪？

对于应用下载类营销而言，用户点击营销后，会对应用采取“下载-安装-激活-注册/次留/付费等应用内行为”的由浅入深的操作。在应用的不同投放阶段，客户希望能够获取不同深度的用户，那些客户希望用户达成的操作就是“转化目标”（例如，倘若客户希望获取到更多激活的用户，那么“激活”就是“转化目标”；倘若客户希望获取到更多的注册用户，那么“注册”就是“转化目标”）。

头条会根据满足转化目标的用户的特征为你找到更多更可能实现转化目标的用户。

但是，头条只能监测到用户在头条端的营销展示和点击等行为，无法监测到用户的应用内行为；客户可以监测到用户的应用内行为，无法监测到用户在头条端的营销展示和点击行为。因此，我们需要“转化跟踪”来实现信息的互通，从而提高营销的优化效果，提高营销投放效率。

2. “应用下载API”与“转化跟踪”是什么关系？

“应用下载API”是一种接口，是实现“转化跟踪的方式之一。

3. “应用下载API”的基本步骤

（1）用户点击营销，巨量引擎将收集到的、全部的营销展示点击等行为通过监测链接发送给客户。前述数据除必要的加密安全处理外，巨量引擎不会进行任何人为处理。

（2）客户收到的信息与自己监测到的用户转化目标相关的信息进行匹配

（3）将匹配成功的转化信息发送至头条的回调地址

4. 应用下载API转化跟踪入口

【跟踪应用直达】

1.调起类营销为什么要做转化跟踪

对于调起类营销而言，使用移动API回传的方式有利于提高调起类营销转化率，降低转化成本，提高跑量速度。可以根据用户在第三方APP的关键行为调整投放群体，将营销投给最可能完成关键操作的用户。

2.我们如何在创建计划的时候使用该功能

在营销目的为“应用”的计划组内可以使用跟踪应用直达，下载方式需要选择落地页方式。

3.创建了转化目标，怎么激活转化目标

在转化列表里头选择检测转化上报，完成下面三个步骤就可以了。1.填写uid，2.获取营销点击数据，3.回传目标用户参数。

4.激活过程是否一定需要客户的技术人员参与？

是的。该对接过程需要客户配合开发，在激活联调过程中，需确认客户能够正常接收用户行为数据，成功匹配转化目标，并成功回传事件。
