# 查询账号流水明细

获取客户或代理商账户流水明细信息。接口规则如下：

1.假设某客户A是代理商下客户，对于A来说，如果流水类型是转账，而收款方是A则意味着是转账加款，如果付款方是A则意味着是转账退款。

2.假设某客户B是直客客户，未在任何代理商下，如果交易流水类型为充值，则收款方和付款方将都显示客户B，如果金额为正则是充值交易，金额为负则是退款交易。

### 请求地址

[https://ad.oceanengine.com/open_api/2/advertiser/fund/transaction/get/](https://ad.oceanengine.com/open_api/2/advertiser/fund/transaction/get/)[可视化调试](https://open.oceanengine.com/tools/visual_debug.html?docId=1696710527205388)[SDK下载](https://open.oceanengine.com/labels/7/docs/1773083358100557)[MCP调用](https://open.oceanengine.com/labels/7/docs/1847394521222599)

### 请求方法

GET

### 请求Header

| 字段 | 类型 | 描述 |
| --- | --- | --- |
| Access-Token 必填 | string | 授权access-token，获取方法见接口文档[【获取Access-Token】](https://open.oceanengine.com/labels/7/docs/1696710505596940) |

### 请求参数

| 字段 | 类型 | 描述 |
| --- | --- | --- |
| advertiser_id 必填 | number | 客户ID |
| start_date 必填 | string | 开始时间，格式YYYY-MM-DD |
| end_date 必填 | string | 结束时间，格式YYYY-MM-DD |
| transaction_type 必填 | string | 流水类型，详见附录[【流水类型】](https://ad.oceanengine.com/openapi/doc/index.html?id=1696710760171535). 允许值: "RECHARGE", "TRANSFER" |
| page | string | 页码. 默认值: 1 |
| page_size | string | 页面数据量. 默认值: 10，最大值：10000 |

注：最大分页深度为100000
