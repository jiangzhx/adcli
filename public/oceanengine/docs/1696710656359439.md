# （已废弃！）转化事件回调服务

### 本文档已废弃！！！请使用新文档：[https://event-manager.oceanengine.com/docs/8650/app_api_docs](https://event-manager.oceanengine.com/docs/8650/app_api_docs)

### 作用

客户在收到头条下发的点击或者展示事件后（通过转化归因-监测API），需要进行用户的归因。在完成归因后，客户需要将相应的转化事件信息传递给字节，并以此计算一次转化。转化事件回传接口，就是用来接受客户自己定义的转化事件的接口，通过调用转化归因-回调API，可以将客户的营销转化相关数据回传给字节。

### 示例

```
xxxxxxxxxx
https://ad.oceanengine.com/track/activate?callback=EJiw267wvfQCGKf2g74ZIPD89-vIATAMOAFCIjIwMTkxMTI3MTQxMTEzMDEwMDI2MDc3MjE1MTUwNTczNTBIAQ==&conv_time=1574835097&event_type=3&os=1&idfa=FCD369C3-F622-44B8-AFDE-12065659F34B&source=mybestcustom
```

### 请求结构

#### 接口地址

```
xxxxxxxxxx
https://ad.oceanengine.com/track/activate/
```

#### 通信协议

支持通过HTTP或HTTPS通道进行请求通信。为了获得更高的安全性，推荐您使用HTTPS通道发送请求。

#### 请求方法

支持HTTP GET 或 POST 方式发送请求，GET方式下请求参数需要包含在请求的URL中，POST方式下请求参数需要包含在请求的Body中。

#### 字符编码

统一采用 UTF-8 的编码格式，url 中的参数如果包含非 ASC 字符，需要进行 url encode 传递

假如我们需要传入一个字段，props = {"aaa":"bbb"}，我们需要对 json 字符串进行 encode

```
xxxxxxxxxx
https://ad.oceanengine.com/track/activate/?callback=xxxx&props={"aaa":"bbb"}
```

### 重要参数

#### Callback

##### 点击事件

我们的点击事件，在下发给客户的时候，会有一个callback字段，这个字段是一个加密字符串，用以标记我们对转化事件的匹配，不可或缺。在客户调用我们接口的时候，需要把这个参数传递回来。

举例说明：

（1）字节在下发点击事件的时候会携带一个 callback 参数 。

```

```

（2）callback参数decode 之后的值是

```
xxxxxxxxxx
EJiw267wvfQCGKf2g74ZIPD89-vIATAMOAFCIjIwMTkxMTI3MTQxMTEzMDEwMDI2MDc3MjE1MTUwNTczNTBIAQ==
```

（3）客户在调用字节的事件回传接口的时候，需要将 callback 参数传递回来

#### 返回值

#### 成功示例

Http code 200 ，标准返回值如下

```
xxxxxxxxxx
1.{
2. "code":0,
3. "ret":0,
4. "msg":"success"
5.}
```

最主要的两个字段是 code 和 msg，code 等于 0 表示请求正常

#### 失败示例

code 不为 0。

msg 会返回错误的提示信息。

#### 常见错误

| 错误信息 | 含义 |
| --- | --- |
| missing params callback | callback_param / callback_url / link 字段没有传入 |
| 参数：outer_event_id无效 | 如果传了非空的outer_event_id，会校验该字段长度不能超过200字符，超过限制则会报错。 |
| 重复的请求，系统已自动过滤 | 如果上报了重复的请求，系统只计算一次转化，重复请求都会被过滤掉。只有请求里带了outer_event_id参数，系统才会去重。 |

### FAQ

一、API回传后，平台没有展示对应的事件量？

请确保event_type传值正确，该接口的event_type需要传「数字」类型的值，如果传成「英文」或其他格式可能会导致接口识别有误，导致事件丢失或事件识别错误等情况。
