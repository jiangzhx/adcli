# 入门指南

感谢您选择巨量引擎开放平台Marketing API，本节将为您介绍开放平台的接入全流程，助您全方位实现自动化营销管理（营销投放、报表分析、创意素材及DMP人群的管理等）

在调用Marketing API前，您必须先注册成为开发者、创建应用并申请对应接口权限。以下将分别为您介绍注册、创建应用、获取权限的具体流程。

请注意重点关注文档中加粗、标注部分的描述。

# 2、如何创建应用程序并申请权限

## 2.1 应用程序分类与作用

创建的应用程序是指基于Marketing API开发的应用，如管理营销的投放平台、数据分析工具、DMP人群自动化管理等。 创建应用程序后可申请开放的功能接口使用权限，对授权客户进行管理操作。

Step1：认证通过成为开发者后，在开发者官网进行登陆并进入「APPID管理」页面，可选择创建的应用类型

Step2：确定应用类型后，进入创建应用程序界面，填写应用名称、图标、申请权限等。

Step3：填写确认提交后，将在一个工作日内审核完成您的应用创建（APPID）申请

## 3.2 应用程序获取调用权限

### 3.2.1 如何获取Access_Token

获取到客户授权返回的auth_code后，可通过【[获取Access Token](https://open.oceanengine.com/labels/7/docs/1696710505596940)】接口获取授权客户对应的AccessToken和RefreshToken

接口请求地址：[https://ad.oceanengine.com/open_api/oauth2/access_token/](https://ad.oceanengine.com/open_api/oauth2/access_token/)

请求参数如下：

| 字段 | 类型 | 描述 |
| --- | --- | --- |
| app_id 必填 | number | 开发者申请的应用APP_ID，可通过应用管理界面查看 |
| secret 必填 | string | 开发者应用的私钥Secret，可通过应用管理界面查看，确保填入secret与app_id对应以免报错 |
| auth_code 必填 | string | 授权码，在授权完成后回调时会提供该授权码，只有10分钟有效期，且只能使用一次确保填入auth_code与app_id对应以免报错 |

请求示例：

```
xxxxxxxxxx
curl --location --request POST 'https://ad.oceanengine.com/open_api/oauth2/access_token/' \
--header 'Content-Type: application/json' \
--data-raw '{
"app_id":123456,
"secret":"xxxx",
"auth_code":"xxxx"
}'
```

返回示例：

```
xxxxxxxxxx
{
"code": 0,
"message": "",
"data": {
"access_token": "xxxxx",
"refresh_token": "xxxxx",
"advertiser_id": 1,
"advertiser_ids": [1,2,3],
"expires_in": 86400,
"refresh_token_expires_in": 604800,
}
```

获取到AccessToken之后，就可以使用AccessToken调用相关功能接口对客户进行管理操作了。 但是请注意，一旦Access_Token失效后，将无法继续调用接口操作，需要刷新获取新的AccessToken才可以继续进行相关调用操作。 Access_Token与Refresh_Token的有效期如下：

### 3.2.2 如何刷新Access_Token

Refresh_Token在有效期内，可以通过接口【[刷新Refresh Token](https://open.oceanengine.com/labels/34/docs/1696710506097679?origin=left_nav)】刷新Access_Token。 请求示例：

```
xxxxxxxxxx
curl --location --request POST 'https://ad.oceanengine.com/open_api/oauth2/refresh_token/' \
--header 'Content-Type: application/json' \
--data-raw '{
"app_id":123456,
"secret":"xxxx",
"grant_type":"refresh_token",
"refresh_token":"xxxx"
}'
```
