# 数据源创建

通过[【数据源文件上传】](https://ad.oceanengine.com/openapi/doc/index.html?id=1696710568556544)接口得到file_path文件路径后，调用当前接口将数据源文件创建成一个数据源，创建成功后会返回一个数据源id，作为数据源的唯一标识。

数据源创建完成后，系统会进行一个数据源解析的过程，将数据源解析成对应的人群包，这个过程大概持续20-60分钟；（同时使用该功能的用户数量多时可能造成解析时间较长，超出60分钟，具体时长将由具体任务量决定，请耐心等待）。 人群包的生成进度可通过[【数据源详细信息】](https://ad.oceanengine.com/openapi/doc/index.html?id=1696710570207247)查询。

### 请求地址

[https://ad.oceanengine.com/open_api/2/dmp/data_source/create/](https://ad.oceanengine.com/open_api/2/dmp/data_source/create/)[可视化调试](https://open.oceanengine.com/tools/visual_debug.html?docId=1696710569089024)[SDK下载](https://open.oceanengine.com/labels/7/docs/1773083358100557)[MCP调用](https://open.oceanengine.com/labels/7/docs/1847394521222599)

### 请求方法

POST

### 请求Header

| 字段 | 类型 | 描述 |
| --- | --- | --- |
| Content-Type 必填 | string | 请求消息类型，允许值：application/json |
| Access-Token 必填 | string | 授权access-token，获取方法见接口文档[【获取Access-Token】](https://open.oceanengine.com/labels/7/docs/1696710505596940) |

### 请求参数

| 字段 | 类型 | 描述 |
| --- | --- | --- |
| advertiser_id 必填 | number | 客户ID |
| data_source_name 必填 | string | 数据源名称, 限30个字符内 |
| description 必填 | string | 数据源描述, 限256个字符内 |
| data_format 必填 | number | 数据格式, 枚举值:"0"：ProtocolBuffer |
| file_storage_type 必填 | string | 数据存储类型, 枚举值:"0"：API |
| file_paths 必填 | string[] | 通过[【数据源文件上传】](https://ad.oceanengine.com/openapi/doc/index.html?id=1696710568556544)接口得到的文件路径，注意：一次上传最多1000个 |
| data_source_type | string | 投放数据源类型，枚举值如下: "UID"， "DID" 选择"UID"时，会根据上传的标识去关联设备上登录的全部用户数据，数据相对量多而广泛； 选择"DID"时，会根据上传的标识仅仅圈选出设备号相同的设备，数据相对量少而精确； 数据源类型为手机号_SHA256 ，仅支持 "UID" 模式； 系统默认选择 UID 模式，请根据自身需要选择对应的模式； |

### 应答参数

| 字段 | 类型 | 描述 |
| --- | --- | --- |
| code | number | 返回码,详见[【附录-返回码】](https://open.oceanengine.com/labels/7/docs/1696710760866831) |
| message | string | 返回信息,详见[【附录-返回码】](https://open.oceanengine.com/labels/7/docs/1696710760866831) |
| data | json | json返回值, 其中data_source_id参数包含唯一标识数据源的字符串 |
| data_source_id | string | 数据源id, 唯一标识数据源的字符串 |
| request_id | string | 请求日志id |
