# 转化回传鉴权接入

## 接入流程

客户切换到鉴权回传转化有3个步骤，分别是新增公钥、测试鉴权、开启鉴权。

## 鉴权方案

使用RS256 (RSASSA-PKCS1-v1_5 using SHA-256) 数字签名算法，以保证消息的完整性、进行认证以及防止否认。

### 必备条件

必须首先通过“新增公钥”接口将PublicKey及对应的credential保存到平台。

用于鉴权的HTTP请求头：

```
xxxxxxxxxx
x-rs256-token: credential=<value>×tamp=<value>&signature=<value>
```

| 字段 | 类型 | 是否必选 | 说明 |
| --- | --- | --- | --- |
| credential | string | 是 | 密钥对label，枚举值："primary"或"backup"。 |
| timestamp | long | 是 | 发出请求的Unix时间戳，单位为秒。它与当前时间相差不能超过 15 分钟。 |
| signature | string | 是 | 生成的签名，计算方法见后文。 |

示例：

```
xxxxxxxxxx
x-rs256-token: credential=primary×tamp=1633766480&signature=vY34PBLBvO07riX2suUHZWhjTw1lQQTbfLiKkmnfEvoQHeuaY2cxomRJkrCQrf1X9A7eKkikLzavxX4+V1QW6qhprAuxOEnA9Jpk9uQY0c18wx5rkiZM4PCIyCFBD6zVpnm4use0QX5lM025EyT08ycInCpraaROXOUKo9EpsH0=
```

### 计算signature

使用RS256算法对StringToSign计算签名，签名结果使用标准Base64编码。

StringToSign格式如下：

```
xxxxxxxxxx
StringToSign=
HTTP_METHOD + '\n' + path_and_query + '\n' + timestamp + '\n' + content_sha256
```

| 字段 | 说明 |
| --- | --- |
| HTTP_METHOD | 请求所使用的大写HTTP方法名。 |
| path_and_query | 串联在一起的请求绝对URI路径和查询字符串。path末尾若没有'/'，需要加上'/'。注意：拼接的query部分需要进行URL编码。 |
| timestamp | 发出请求的Unix时间戳，单位为秒。它与当前时间相差不能超过 15 分钟。 |
| content_sha256 | 请求正文的标准 base64 编码的 SHA256 哈希。 即使没有正文，也必须提供它。 base64_standard_encoding(SHA256(body)) |

示例：

```
xxxxxxxxxx
StringToSign=
"GET" + '\n' +                                 // VERB
"/track/activate/?event_type=324&callback=CNXUIhCpEhgAOABCFTEyMzEzMTIzMTJqbGtqa2wxNDMxM5ABAA==" + '\n' +                     // path_and_query
"1633766480" + '\n' +                          // timestamp
"8ppEi3gHRb8uEGZ/RsRCsQLnXnakah//lpZBhmIlq1Y=" // content_sha256
```

RS256签名示例：

```
xxxxxxxxxx
String rs256(PrivateKey privateKey, String stringToSign) throws Exception {
final Signature s = Signature.getInstance("SHA256withRSA");
s.initSign(privateKey);
s.update(stringToSign.getBytes(StandardCharsets.UTF_8));
byte[] sig = s.sign();
String signature = Base64.getEncoder().encodeToString(sig);
return signature;
}
```

*

private static String calcToken(HttpUriRequest request, String credential) throws Exception {
// get stringToSign
String methodName = request.getRequestLine().getMethod().toUpperCase();
URI uri = request.getURI();
String pathAndQuery = uri.getPath();
// 在path末尾加上'/'
if (pathAndQuery == null) {
pathAndQuery = "/";
} else if (!pathAndQuery.endsWith("/")) {
pathAndQuery += "/";
}
if (uri.getRawQuery() != null) {
pathAndQuery = pathAndQuery + "?" + uri.getRawQuery();
}
long timestamp = Instant.now().getEpochSecond();
String contentHash = buildContentHash(request);
String stringToSign = String.format("%s\n%s\n%s\n%s", methodName, pathAndQuery, timestamp, contentHash);
// System.out.println("stringToSign: " + stringToSign);
​
// get signature
String keyFile = String.format("./src/main/resources/pkcs8-%s.pem", credential);
PrivateKey privateKey = PemUtils.readPrivateKeyFromFile(keyFile, "RSA");
String signature = rs256(privateKey, stringToSign);
​
// x-rs256-token: credential=<value>×tamp=<value>&signature=<value>
return String.format("credential=%s×tamp=%s&signature=%s", credential, timestamp, signature);
}
​
static class PemUtils {
​
public static PrivateKey readPrivateKeyFromFile(String filepath, String algorithm) throws IOException {
byte[] bytes = parsePemFile(new File(filepath));
return PemUtils.getPrivateKey(bytes, algorithm);
}
​
private static byte[] parsePemFile(File pemFile) throws IOException {
if (!pemFile.isFile() || !pemFile.exists()) {
throw new FileNotFoundException(String.format("The file '%s' doesn't exist.", pemFile.getAbsolutePath()));
}
PemReader reader = new PemReader(new FileReader(pemFile));
PemObject pemObject = reader.readPemObject();
byte[] content = pemObject.getContent();
reader.close();
return content;
}
​
private static PrivateKey getPrivateKey(byte[] keyBytes, String algorithm) {
PrivateKey privateKey = null;
try {
KeyFactory kf = KeyFactory.getInstance(algorithm);
EncodedKeySpec keySpec = new PKCS8EncodedKeySpec(keyBytes);
privateKey = kf.generatePrivate(keySpec);
} catch (NoSuchAlgorithmException e) {
System.out.println("Could not reconstruct the private key, the given algorithm could not be found.");
} catch (InvalidKeySpecException e) {
System.out.println("Could not reconstruct the private key");
}
​
return privateKey;
}
}

/**
* 计算请求体 sha-256 hash
*
* @param request http 请求
* @return 请求体 sha-256 hash
* @throws IOException on exception
*/
private static String buildContentHash(HttpUriRequest request) throws IOException {
String content = "";
if (request instanceof HttpEntityEnclosingRequest) {
try {
StringWriter writer = new StringWriter();
IOUtils.copy(((HttpEntityEnclosingRequest) request).getEntity().getContent(), writer,
StandardCharsets.UTF_8);
​
content = writer.toString();
} finally {
((HttpEntityEnclosingRequest) request).getEntity().getContent().close();
}
}
​
byte[] digest = new DigestUtils("SHA-256").digest(content);
return Base64.getEncoder().encodeToString(digest);
}

* @param privateKey   私钥
* @param stringToSign 要签名的字符串
* @return 签名
* @throws Exception on exception
*/
private static String rs256(PrivateKey privateKey, String stringToSign) throws Exception {
final Signature s = Signature.getInstance("SHA256withRSA");
s.initSign(privateKey);
s.update(stringToSign.getBytes(StandardCharsets.UTF_8));
byte[] sig = s.sign();
return Base64.getEncoder().encodeToString(sig);
}

// calcToken 计算x-rs256-token
// credential为密钥label：取值为"primary"或"backup"
func calcToken(req *http.Request, credential string) (string, error) {
// get stringToSign
method := req.Method
pathAndQuery := req.URL.Path
// 在path末尾加上'/'
if pathAndQuery == "" || pathAndQuery[len(pathAndQuery)-1] != '/' {
pathAndQuery = pathAndQuery + "/"
}
if req.URL.RawQuery != "" {
pathAndQuery = pathAndQuery + "?" + req.URL.RawQuery
}
​
timestamp := time.Now().Unix()
​
var content []byte
​
if req.Body != nil {
body, err := ioutil.ReadAll(req.Body)
if err != nil {
return "", err
}
req.Body = ioutil.NopCloser(bytes.NewBuffer(body))
​
content = body
}
​
contentHash := getContentHashBase64(content)
​
stringToSign := fmt.Sprintf("%v\n%v\n%v\n%v", strings.ToUpper(method), pathAndQuery, timestamp, contentHash)
​
// get signature
keyFile := fmt.Sprintf("./src/main/resources/pkcs8-%v.pem", credential)
privateKey, err := readPrivateKeyFromFile(keyFile)
if err != nil {
return "", err
}
​
rng := rand.Reader
​
hashed := sha256.Sum256([]byte(stringToSign))
​
sigBytes, err := rsa.SignPKCS1v15(rng, privateKey, crypto.SHA256, hashed[:])
if err != nil {
return "", err
}
signature := base64.StdEncoding.EncodeToString(sigBytes)
​
return fmt.Sprintf("credential=%v×tamp=%v&signature=%v", credential, timestamp, signature), nil
}

func getContentHashBase64(content []byte) string {
hasher := sha256.New()
hasher.Write(content)
return base64.StdEncoding.EncodeToString(hasher.Sum(nil))
}

// parseRSAPrivateKeyFromPEM Parse PEM encoded PKCS1 or PKCS8 public key
func parseRSAPrivateKeyFromPEM(key []byte) (*rsa.PrivateKey, error) {
var err error
​
// Parse PEM block
var block *pem.Block
if block, _ = pem.Decode(key); block == nil {
return nil, err
}
​
// Parse the key
var parsedKey interface{}
if parsedKey, err = x509.ParsePKCS8PrivateKey(block.Bytes); err != nil {
return nil, err
}
​
var pkey *rsa.PrivateKey
var ok bool
if pkey, ok = parsedKey.(*rsa.PrivateKey); !ok {
return nil, fmt.Errorf("not *rsa.PrivateKey type")
}
​
return pkey, nil
}

## 管理接口

管理接口的调用都需要通过[Marketing API（OAuth 2.0）](https://open.oceanengine.com/doc/index.html?key=ad&type=api&id=1696710502277120)授权。

### 新增公钥

客户将公钥保存到平台。

为支持切换密钥的场景，客户最多可使用两对公/私钥对，两对的label约定为“primary”、“backup”。

为确保客户使用了正确的公钥格式和签名算法，在新增公钥时有一步验签的动作，客户使用私钥对指定文本“BYTEDANCE”签名。

请求地址

[https://ad.oceanengine.com/open_api/2/event_manager/auth/add_public_key](https://ad.oceanengine.com/open_api/2/event_manager/auth/add_public_key)

POST

| pubkey | string | 是 | RSA公钥（1024位）文本内容，格式为[X.509 Certificate Subject Public Key Info](https://datatracker.ietf.org/doc/html/rfc7468#section-13)，可使用openssl生成，详见附录。注意：使用openssl生成的公钥是多行文本，需要使用换行符“\n”将多行文本转为单行。 |
| signature | string | 是 | 使用RS256签名算法和私钥对字符串“BYTEDANCE”签名后的结果，可使用openssl计算，详见附录 |

### 开启鉴权

客户开启鉴权。

鉴权关闭状态下，鉴权失败的转化数据不会被丢弃，同时返回鉴权失败原因，方便客户调试定位。开启鉴权后，鉴权失败的转化数据将被丢弃。平台会缓存客户的鉴权开启状态，缓存时长约3分钟，这意味着，最长会有3分钟的过渡时间。

客户确认上线后有成功回传转化，且没有鉴权失败（观察24h以上，最长不超过一周)之后，可请求“开启鉴权”接口，平台检查是否满足开启条件，如果满足则开启，否则开启失败。

### 通过命令行方式生成密钥对并签名

1. 生成RSA密钥对

生成1024位RSA私钥private-key.pem：

```
xxxxxxxxxx
openssl genrsa -out private-key.pem 1024
```

备注：可通过如下命令将私钥private-key.pem转换为[pkcs8](https://en.wikipedia.org/wiki/PKCS_8)格式pkcs8.pem：

openssl pkcs8 -topk8 -inform PEM -in private-key.pem -outform pem -nocrypt -out pkcs8.pem

从私钥中提取公钥public-key.pem：

```
xxxxxxxxxx
openssl rsa -in private-key.pem -pubout -outform PEM -out public-key.pem
```

查看公钥文本内容：

```
xxxxxxxxxx
cat public-key.pem
-----BEGIN PUBLIC KEY-----
MIGfMA0GCSqGSIb3DQEBAQUAA4GNADCBiQKBgQDSLWmtc3edSzIp7d2QoHCnjNNx
cjGURr/y43fuCAjFZFhxYiIhrNF0p5gmI/c3B8o05aTf8BDXU3BFjZupyzTHoRKr
Hg4hoPvn2EH9UfxICp1bCtIcwe1a4rjCBd/CAaF6DCKp19Pvci1+e8xTJXU0ERVM
6K5yu6aixC2y0NUMdwIDAQAB
-----END PUBLIC KEY-----
```

2. 计算signature

使用私钥private-key.pem计算字符串"BYTEDANCE"的签名：

```
xxxxxxxxxx
echo -n "BYTEDANCE" | openssl dgst -sha256 -binary -sign private-key.pem  | openssl enc -base64 | tr -d '\n'
```

上面命令的输出结果即生成的签名：

```

```
