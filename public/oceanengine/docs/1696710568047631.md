# DMP人群管理

## 概念介绍

DMP（Data Management Platform，数据管理平台）人群管理：

在这个功能模块下，用户可以实现创建、管理数据源和人群包的功能。

数据源：

API提供第三方数据上传，用户可以按照特定格式上传包含设备id/用户id的数据信息，这些数据称之为“数据源”，数据源经过解析后，生成对应的人群包。

人群包：

什么是人群包：了解什么是人群包，首先需要知道“人群定向”的概念，当我们创建营销计划时，可以针对这个营销计划设置“用户定向”，用户定向包含多种定向方式（地域，关键词，app行为，自定义人群等），人群定向是其中一种，在选择目标人群时，我们可以使用人群包快速定向/排除特定人群。

投放限制：

通过 Open API 创建的人群包，只能用于效果营销投放，不能用于投放，因为这类人群包数据可被更新，无法锁定库存。

如果需要将通过 Open API 创建的人群包用于投放，可以在 DMP 界面创建人群包，将通过 Open API 创建的人群包作为子包包含进去，这样会把那一天的数据固化下来用来锁定库存，进而用来给投放。

在这种情况下如果人群包进行数据更新，固化下来的数据也仍然会生效，所以需要谨慎使用。

## 流程简述

步骤一、创建人群包（通过接口同步人群包数据）

步骤二、发布和推送人群包

步骤三、更新人群包

## 相关问题Q&A

Q：数据源和人群包是一一对应的吗？ A：每个数据源都可以被解析成一个人群包，但人群包还可以进行进一步的运算/拓展来生成新的人群包但不会改变数据源。同时，用户也可以通过创建规则人群包来不通过数据源获得人群包。

Q：如何使用人群包？ A：当人群包成功发布并推送给需要的客户账号后，该客户创建计划时可以在“用户定向——人群定向”处定向/排除选定的人群包。同时DMP数据管理平台提供了更加丰富的功能，包括针对人群包的“人群洞察”和提供给用户快速添加需要的人群包的“标签市场”，如有需要，请移步营销投放平台。

Q：人群包生效是否意味着可以使用？ A：人群包状态为已生效只是表示该人群包经过了计算，用于投放还需要经过发布和推送的过程。delivery_status:人群包可投放状态 披露了人群包的发布，推送状态。当人群包为已发布且已推送才可正式使用。

Q：人群包的发布和人群包的推送有什么区别？ A：发布意味着让一个人群包进入投放系统，推送指给予指定同主体客户使用该人群包的权限，正式使用人群包需要完成发布，推送两个步骤，缺一不可。

Q：再次发布/推送人群包的应用场景是什么? A：每个人群包都都有一个人群包id作为唯一标识，这个id是在数据源解析成人群包后生成的，可以通过[【数据源详细信息】](https://ad.oceanengine.com/openapi/doc/index.html?id=1696710570207247)获取。 [【数据源更新】](https://ad.oceanengine.com/openapi/doc/index.html?id=1696710569591823)只改变人群包内容，不改变人群包id，所以只需要再次发布，不需要推送。

Q：调用接口后返回success，就代表操作成功了吗？ A：接口返回success，不代表人群包是success的，需要关注data.change_logs.message字段 可自查的失败原因： 1.文件是否换行，有过case把所有的ID都写到了一行上 2.文件校验失败，通常是ID类型和ID值匹配错误 或者 PB版本错误。正确的参考文档中Protocol Buffer 指定id类型部分。同时可以使用 [文档](https://open.oceanengine.com/labels/7/docs/1696710761459712)中校验demo进行自查。

Q：字节 App 端能保证采集到设备 ID（比如 OAID，IDFA，IMEI 等）吗？如果采集不到，会怎么处理？

A：出于合规的原因，字节并不能保证一定能采集到所有的设备 ID，在采集不到的情况下，如果是排除场景，会默认采集不到的设备不在排除范围内，如果是定向场景，则会认为没有采集到的设备不在定向范围内。这是无法避免的现象，请使用者自行斟酌投放策略。

## 限制

| 功能 | 数量上限 |
| --- | --- |
| 数据源创建 | 每个客户账号一天只能创建100次数据源 |
| 数据源更新 | 每一个数据源一天只能更新50次 |
| 人发布群包 | 每个人群包每24小时只可发布一次 |
| 人群包数量限制 | 每个客户最多只能创建 400 个人群包 |
| 人群包规范 | 人群包的名字需要小于等于 30 个字符 |
