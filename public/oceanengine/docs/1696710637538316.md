# 创建卡券

创建青鸟线索通卡券接口

### 请求地址

[https://ad.oceanengine.com/open_api/2/clue/coupon/create/](https://ad.oceanengine.com/open_api/2/clue/coupon/create/)[可视化调试](https://open.oceanengine.com/tools/visual_debug.html?docId=1696710637538316)[SDK下载](https://open.oceanengine.com/labels/7/docs/1773083358100557)[MCP调用](https://open.oceanengine.com/labels/7/docs/1847394521222599)

### 请求方法

POST

### Header

| 字段 | 类型 | 描述 |
| --- | --- | --- |
| Access-Token 必填 | string | 授权access_token，可以通过[【获取Access token】](https://ad.oceanengine.com/openapi/doc/index.html?id=1696710506574848)接口获取 |
| Content-Type 必填 | string | 请求消息类型 允许值: "application/json" |

### 请求参数

| 字段 | 类型 | 描述 |
| --- | --- | --- |
| advertiser_id 必填 | number | 客户ID |
| activity_type 必填 | string | 卡券类型，可选值: DIRECT_NEED_PHONE：直接发券，收集手机号，need_phone须为true DIRECT_NOT_NEED_PHONE ：直接发券，不收集手机号，need_phone须为false 并且 BindFormId会被忽略 activity_type和need_phone传入不匹配会报错 |
| coupon | object | 卡券信息 |
| title 必填 | string | 活动标题，不超过15字 |
| deliver_start 必填 | string | 活动开始时间，格式：%Y-%m-%d， 小于结束时间 若小于当天则立即生效 |
| deliver_end 必填 | string | 活动结束时间，格式：%Y-%m-%d，大于当前时间，限制最多一年 |
| global_limit | object | 总库存量限制 |
| day_limit | number | 每日投放的总库存，不填或0表示不限制 |
| total_limit | number | 总投放库存限制，不填或0表示不限制 |
| user_limit | object | 单个用户的库存量限制 |
| day_limit | number | 用户每日可参与库存限制，不填或0表示不限制 |
| total_limit | number | 用户可参与总库存限制，不填或0表示不限制 |
| resource_list 必填 | object[] | 优惠券列表 目前只支持1个优惠券，传入多于一个会报错 |
| title 必填 | string | 优惠券标题，不超过15字 |
| merchant_name 必填 | string | 商户名称，不超过15字 |
| valid_days | number | 有效时间段，从领取日开始计算，最大值不超过365 设置此值后，valid_start和valid_end会被忽略，不填或0表示不限制 |
| valid_start | string | 有效期开始时间 格式：%Y-%m-%d, 起始时间小于当日表示立即生效 |
| valid_end | string | 有效期结束时间 格式：%Y-%m-%d, 最大值距当前时间不得超过一年。 valid_days 和 [valid_start、valid_end] 至少填一个 valid_end >= deliver_end |
| code_type 必填 | string | 券码类型，可选值: API：合作方api实时传入的券码 COMMON：通用码，固定值，不支持核销 MERCHANT：商家上传券码，需调用上传券码接口上传券码文件 PLATFORM：平台产生券码 |
| code | string | 通用券码类型时设置的券码，4-30位，只支持字母、数字。 通用券码类型时必填 |
| condition 必填 | string | 使用须知，长度不超过50字 |
| notification 必填 | string | 领取须知，长度不超过500 |
| head_image_url 必填 | string | 卡券头图地址，即优惠券背景图，支持jpg、jpeg、png文件,分辨率为686*176,大小1m以下 |
| logo_img_url 必填 | string | 商家logo地址 |
| service_num 必填 | string | 客服电话 不允许填写的特殊电话列表：119,110,120,122,12110,114,11185,12348,12395,12121,12117,999,,95119,112,12306,95598,12315,12365,12369,12345,12318,12358,12310,12333,12320 |
| stock | number | 优惠券的库存，0表示不限制 |
| use_type 必填 | string | 核销类型 OFFLINE：用户线下核销 LINK：用户使用时直接跳转新链接 |
| link | string | use_type为LINK时必填，跳转链接 |
| industry_type 必填 | string | 行业列表，可选值: ENTERTAINMENT：生活娱乐 FINANCIAL：互联网金融 FOOD：美食 GAME：游戏 TICKET：休闲票务 OTHER：其他 |
| store_ids | number[] | 可用门店ID列表，建议不超过500 |
| min_amount | number | 最低消费金额，单位为分，如：500 表示500分，即5元。0或不填代表不限制 |
| resource_type 必填 | string | 优惠券类型。详见[【附录-枚举值】](https://ad.oceanengine.com/openapi/doc/index.html?id=1696710760171535)，可选值: COMMON：通用券 DISCOUNT：折扣券 FULL：满减券 GAME：游戏礼包 PHYSICAL：实物券 |
| gift_list | object[] | 游戏礼包列表，优惠券类型为游戏礼包时必填 建议数量：1-5个 |
| count | number | 礼品数量，优惠券类型为礼包码时必填，大于0 |
| icon_url | string | 礼品icon地址，优惠券类型为礼包码时必填 |
| name | string | 礼品名称，优惠券类型为礼包码时必填，不超过12字 |
| ios_link | string | ios端下载链接，优惠券类型为礼包码时必填 请填写有效连接，否则会造成无法跳转 |
| android_link | string | 安卓端下载链接，优惠券类型为礼包码时必填 请填写有效连接，否则会造成无法跳转 |
| discount | number | 折扣券的折扣，值为1到99，如：1代表0.1折，90代表9折。优惠券类型为折扣券时必填,不填会报错’invalid amount‘ |
| max_amount | number | 优惠券类型为折扣券时的最大减免金额，单位为分，如：500 表示500分，即5元。0或不填代表不限制 |
| relief_amount | number | 满减券的减免金额，单位为分，如：500 表示500分，即5元。优惠券类型为满减券时必填 |
| need_phone | number | 是否收集手机号 1：收集 0：不收集(默认) |
| need_sms_verify | number | 是否对手机号进行验证 1：验证 0：不验证 need_phone=0时，该字段无意义 |
| bind_form_id | number | 绑定的表单ID。仅need_phone=1时，支持设置此值. 不设置此值时，系统会默认为当前的卡券创建一个用来收集手机号的表单 使用者也可以明确指定要绑定的表单，此表单的验证类型会覆盖need_sms_verify的值 使用者指定的表单必须有且仅有姓名和电话两个字段 |
