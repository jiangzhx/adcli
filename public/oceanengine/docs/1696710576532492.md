# DPA投放相关接口

如果你要投放产品目录营销（以下简称DPA）首先需要有改投放的权限，可联系对接运营同学了解，创建DPA计划组、计划、创意与其他类型的营销有着部分区别，具体如下：

### 项目投放接口

营销目的landing_type新增DPA枚举，如果要创建DPA计划组则campaign/create接口的landing_type字段需要传DPA值。

针对DPA营销，其他类型的营销无需使用：

| 字段 | 类型 | 备注 |
| --- | --- | --- |
| product_platform_id | int | 商品库id，(ID由查询可用查询可用商品库接口得到), 当营销目的landing_type=DPA时必填 |
| category_type | string | 商品投放范围，NONE不限，CATEGORY按分类，PRODUCT按商品id |
| dpa_categories | list<int> | 分类列表，投放范围为分类时必填 |
| dpa_products | list<int> | 商品列表，投放范围为商品时必填 |
| dpa_adtype | string | dpa营销类型，落地页，应用下载二选一，DPA_LINK落地页，DPA_APP应用下载 |
| params_type | string | 落地页链接类型，DPA商品库链接，CUSTOM自定义链接 |
| dpa_external_urls | list<string> | 落地页链接列表，自定义链接必填，目前只能填一个 |
| dpa_external_url_field | string | 商品库链接必填，规则详见 ad维度落地页链接规则 |
| dpa_open_url_type | string | 直达链接类型，NONE无，DPA商品库链接，CUSTOM自定义链接 |
| dpa_open_urls | list | 直达链接列表，自定义链接必填，目前只能填一个 |
| dpa_open_url_field | string | 商品库链接必填，规则详见 ad维度直达链接规则 |
| dpa_product_target | list<dpaproducttarget> | 商品库定向 |
| user_type | list<string> | 自定义定向 |
| external_url_params | string | H5地址参数(DPA营销目的特有,在填写的参数后面添加"=urlencode(开放平台提供的h5链接地址）"，其中urlencode(开放平台提供的h5链接地址）替换为商品库中的h5地址encode的结果) |
| open_url_params | string | 直达链接参数((DPA营销目的特有,在“产品库中提取的scheme地址"后面追加填写的参数) |
| dpa_local_audience | number | 是否自定义商品定向((DPA营销目的特有,1表示自定义) |

### DPA产品投放条件字段格式

| 字段 | 类型 | 备注 |
| --- | --- | --- |
| title | string | 筛选条件，规则详见 **DPA计划监测直达链接规则** |
| type | string | 条件类型 |
| rule | string | 规则针对 type为int, long, double, floatrule可选：=, !=, >, >=, <, <=针对 type为 stringrule可选：=, !=, contain（包含）, exclude（不包含） |
| value | string | 值，可以输入多个，用\n分隔 |

### DPA人群行为字段格式

| 字段 | 类型 | 备注 |
| --- | --- | --- |
| code | int | 人群行为，通过人群行为接口获得 |
| days | int | 天数，可选1，7，14，28，60，90，120，180 |

### DPA营销落地页链接规则

使用商品库元信息获得，用media_type=4进行过滤 使用返回中title = targetUrlMobile, brandUrlMobile, shopKeeperUrlMobile的 field_group字段即可。 如此处选择使用商品库所含链接，如下： 对应的参数是

```
xxxxxxxxxx
{
"params_type": "DPA",
"dpa_external_url_field": "targetUrl", # 使用field_group中的值
# "dpa_external_urls": [] 这个参数不需要
}
```

### DPA计划直达链接规则

使用商品库元信息获得，用media_type=4进行过滤， 使用返回中title = targetUrlIOSApp、targetUrlAndroidApp、brandUrlIOSApp、brandUrlAndroidApp、shopKeeperUrlIOSApp、shopKeeperUrlAndroidApp的 field_group字段即可。

例如这里，可以使用传targetUrl 如果是 安卓应用下载，可以使用title为targetUrlAndroidApp的field_group 如果是 IOS应用下载，可以使用title为targetUrlIOSApp的field_group 如果直达链接选择商品库所含链接，同落地页链接规则，使用参数如下：

```
xxxxxxxxxx
{
"dpa_open_url_type": "DPA",
"dpa_open_url_field": "targetUrl", # 这个参数规则同上
# "dpa_open_urls": []
}
```

### DPA计划监测直达链接规则

使用商品库元信息获得，用indexable=1进行过滤 使用 返回中 的 title和type，rule和value根据规则填写

### 营销创意投放接口

针对DPA创意，创意创建接口和创意更新接口的创意拼装方式与普通创意将有比较大的区别，DPA创意使用的是创意模板的方式新增了创意ID，并且DPA的创意动态词包用的是新字段，具体如下：

| 字段 | 类型 | 备注 |
| --- | --- | --- |
| dpa_dict_ids | list<int> | dpa词包列表，对应的标题中应该用$$将替换词包围，格式如下：""。词包名$ 词包名$ |
| template_id | int | 模板id，注意点：仅大图小图组图能够使用模板，创意类型需要和模板类型相匹配，比如小图只能用小图的模板。 |

### DPA创意模板传参方式

由于平台侧是通过前端提供界面，对于客户来说比较好理解，客户填完表单之后会动态生成一个json参数给后端，但是对于API接入方来说，不同的模板的参数都不相同，其中有的允许修改有的不允许修改，理解成本很高，因此此处只传一个template_id。

示例 url： /open_api/2/dpa/meta/get/ 参数：

```
xxxxxxxxxx
{
"advertiser_id": fields.Int(required=True),
"platform_id": fields.Int(required=True,
validate=with_msg(partial(validate_min_number, min_number=1), u'platform_id大于0')),
"indexable": fields.Int(),
"extractable": fields.Int(),
"industry": fields.Int(),
"status": fields.Int(),
"mediaType": fields.Int()
}
```
