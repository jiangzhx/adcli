# 数据源详细信息

通过数据源id，查询该数据源相关信息和其对应的人群包信息。数据源解析完成后会返回default_audience：数据源对应的人群包相关信息，其中人群包ID是该人群包的唯一标识。解析未完成不返回default_audience：数据源对应的人群包相关信息。

数据源解析成人群包后，调用该接口会返回该数据源对应的人群包的人群包id，作为该人群包的标识； 数据源在创建后被解析成人群包需要20-60分钟。高峰期可能造成解析时间延长，请您耐心等待； 本接口可以批量查询多个数据源的信息。其中应答参数是按create_time：数据源创建时间倒序返回的。如果您有数据处理的需求，可以参考此排序逻辑。

### 请求地址

[https://ad.oceanengine.com/open_api/2/dmp/data_source/read/](https://ad.oceanengine.com/open_api/2/dmp/data_source/read/)[可视化调试](https://open.oceanengine.com/tools/visual_debug.html?docId=1696710570207247)[SDK下载](https://open.oceanengine.com/labels/7/docs/1773083358100557)[MCP调用](https://open.oceanengine.com/labels/7/docs/1847394521222599)

### 请求方法

GET

### 请求Header

| 字段 | 类型 | 描述 |
| --- | --- | --- |
| Access-Token 必填 | string | 授权access-token，获取方法见接口文档[【获取Access-Token】](https://open.oceanengine.com/labels/7/docs/1696710505596940) |

### 请求参数

| 字段 | 类型 | 描述 |
| --- | --- | --- |
| advertiser_id 必填 | number | 客户ID |
| data_source_id_list 必填 | string[] | 数据源ID列表 一次最多传400个数据源id |

| request_id | string | 请求日志id |
