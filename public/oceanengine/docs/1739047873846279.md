## 2.库表命名

| 表名 | 数据用途 | 数据粒度 | 推送触发 | 更新频次 | 取数方法 |
| --- | --- | --- | --- | --- | --- |
| lianshan_realtime_{当周第一天日期} | 实时账户/计划组/计划/创意小时累计报表 实时账户/计划组/计划/创意日累计报表等 | 创意 × 营销展位 (creative_id × inventory) | 任一指标发生变化才推送 指标数据没有变化不推送 | 3min一更新 当前小时指标数据覆盖更新 | 1.创意/计划/计划组/账号分别聚合(group by)即可 2.日/周/月数据对聚合好的数据求和(sum)即可 3.整点后15分钟可以获取到之前整点的稳定数据 4.次日9点可以获取到前一日的持久化修正数据 |

库表命名规则

库命名：“lianshan_rds_{订阅任务ID} ”，例如lianshan_rds_1731339863784456”

表命名：“lianshan_realtime_{当周第一天日期}”，例如lianshan_realtime_20220425，按周分表

| 名称 | 类型 | 说明 | 业务含义 |
| --- | --- | --- | --- |
| id | bigint | 主键id | 数据推送记录id |
| 数据指标1 |  |  |  |
| ... |  |  |  |
| 数据指标n |  |  |  |
| stat_time | datetime | 数据产生时间 | 报表数据的产生时间 |
| create_time | datetime | 记录创建时间 | 数据推送的记录时间 |
| update_time | datetime | 数据更新时间 | 已落库记录覆盖更新后creat_time不变，update_time会更新至最新记录落库时间 |
