# 创建营销链路转化

# 1. 概述

## 1.1. 背景

在广告系统升级版（3.0 系统）中进行广告创建时，将新增对广告转化「营销链路」的表达。 因此广告主在投放前，需在「转化归因」模块完成营销链路的转化规则创建工作，并按要求完成数据的回传及联调。

## 1.2. 概念

| 名词 | 说明 |
| --- | --- |
| 营销链路 | 营销链路是指用户在营销活动中，从广告点击到完成转化所经过的完整路径，由承接转化的营销载体和多个行为节点序列组成。广告主通过在广告创编中选择营销链路，精准表达营销标的和转化路径。 |
| 转化分类 | 转化分类对应广告投放的「营销载体」概念，是营销链路的优化目标（或生意目标）发生的场所。例如某次投放的优化目标是促进用户在某小程序上完成交易，那么在为该广告准备营销链路资产时，需要选择该小程序作为转化分类。 |
| 优化目标 | 在同一条营销链路中，选择不同的优化目标时，会有不同的数据回传要求，需满足要求后才可在投放时选择对应优化目标 |
| 上报节点 | 在营销链路的节点序列中，广告主需回传对应数据的节点，即上报节点。上报节点包括所选优化节点要求的必传节点和推荐回传节点，所选的上报节点均需在联调工具中完成数据联调。 |

# 2. 指引

## 2.1. 步骤 1：选择营销链路

### 2.1.1. 概述

腾讯广告按照各行业的业务实际情况，已预置了一批营销链路模板，不同模板的区别在于：用户经过的节点序列（即行为的类型和载体）不同。因此在创建营销链路资产前，需先通过 conversion_links/get 接口，获取可用的营销链路模板，并根据实际需求选择对应的节点序列。 在选择营销链路模板时，可根据下述信息，进行链路的区分和选择：

转化分类：按照回传数据的客户端类型（如 iOS APP、微信小程序），查询适用的营销链路

优化目标：按照广告投放的优化目标组合，查询适用的营销链路（可以通过[查询优化目标权限](https://developers.e.qq.com/v3.0/docs/api/optimization_goal_permissions/get)接口查询是否有对应优化目标权限）

#### 转化分类

| 枚举值 | 含义 |
| --- | --- |
| ANDROID | 安卓应用 |
| IOS | iOS 应用 |
| QUICK_APP | 快应用 |
| WEB | 网页 |
| WECOM | 企业微信 |
| WECHAT_CHANNELS | 微信视频号 |
| WECHAT_MINI_PROGRAM | 微信小程序 |
| WECHAT_OFFICIAL_ACCOUNT | 微信公众号 |
| WECHAT_MINI_GAME | 微信小游戏 |
| QQ_MINI_GAME | QQ 小游戏 |

#### 优化目标类型

| 字段 | 含义 |
| --- | --- |
| optimization_goal | 优化目标 |
| deep_optimization_goal | 深度优化目标 |
| forward_link_assist | 助攻行为目标 |
| deep_advanced_optimazation_goal | 深度辅助目标 |
| deep_conversion_optimization_goal | 深度优化 ROI 目标 |
| deep_advanced_roi | 深度辅助 ROI 目标 |

## 2.2. 步骤 2：创建转化归因

### 2.2.1. 概述

创建转化归因时，需基于选择的营销链路，重点配置如下信息：

转化分类和对应 id：根据所选的转化分类，填写具体的 id（如安卓应用 id、IOS 应用 id、小游戏 id 等）

优化目标：填写广告投放的优化目标信息（选择不同优化目标，会有不同的数据上报节点）

上报节点：根据所选优化目标的要求，选择需上报数据的营销链路节点（每个上报节点有对应的数据来源、回传要求和优化方式）

归因方式：选择优化目标对应的归因方式（如点击归因、曝光归因、激活归因等）

监测链接组：从 DataNexus 中配置的监测链接组中，选择后续广告点击/曝光数据下发使用的监测链接组

#### 数据来源

| 数据来源 | 字段 |
| --- | --- |
| 平台上报 | DATA_SOURCE_PLATFORM |
| 广告主上报 | DATA_SOURCE_ADVERTISER |

#### 回传要求

| 回传要求 | 字段 |
| --- | --- |
| 必须上报 | NODE_SEQS_TYPE_REQUIRE |
| 推荐上报 | NODE_SEQS_TYPE_RECOMMEND |
